# Web Servers: Concepts, Types.

## Introduction

A **web server** (also known as an **HTTP server**) is software that delivers web content to users through the internet or a private network. Whenever a user enters a URL into a browser, the browser sends a request to a web server, which then responds with the requested resource.

At its most basic level, a web server makes files stored on a computer accessible to other computers through a web browser. These files can include:

- HTML pages
- Images
- Videos
- CSS files
- JavaScript files
- Documents

Modern web servers do much more than serve files. They can generate dynamic content, process user requests, connect to databases, host APIs, and run web applications.

Understanding web servers is fundamental for system administrators, web developers, cloud engineers, and DevOps professionals.

---

# What is a Web Server?

A web server consists of two components:

1. **Hardware**
   - The physical or virtual machine that stores website files.

2. **Software**
   - The application that listens for HTTP requests and delivers responses.

Examples of web server software include:

- Apache HTTP Server
- Nginx
- LiteSpeed
- Caddy
- Lighttpd
- Microsoft IIS

---

# How Web Servers Work

The basic request-response cycle works as follows:

1. User enters a URL in a browser.
2. DNS resolves the hostname to an IP address.
3. Browser sends an HTTP request.
4. Web server receives the request.
5. Server locates the requested resource.
6. Server sends an HTTP response.
7. Browser renders the content.

```text
Browser → HTTP Request → Web Server
Browser ← HTTP Response ← Web Server
```

Example:

```text
http://example.com/index.html
```

Components:

| Component | Description |
|------------|-------------|
| http | Protocol |
| example.com | Hostname |
| index.html | Resource |

---

## DNS

The Domain Name System (DNS) translates human-readable hostnames into IP addresses.

Example:

```text
www.example.com
```

may resolve to:

```text
93.184.216.34
```

Without DNS, users would need to remember IP addresses instead of domain names.

# Types of Web Servers

Web servers can be classified based on how they process and deliver content.

## 1. Static Web Servers

A static web server delivers files exactly as they exist on disk.

### Characteristics

- Fast
- Simple
- Low resource usage
- No server-side processing

### Example

If the server contains:

```html
<h1>Hello World</h1>
```

every user receives the exact same content.

### Typical Files

- HTML
- CSS
- JavaScript
- Images
- PDFs

### Use Cases

- Documentation websites
- Portfolio websites
- Landing pages
- Blogs generated by static site generators

### Advantages

- High performance
- Easy to host
- Improved security
- Lower server requirements

### Disadvantages

- Limited interactivity
- Cannot generate personalized content

---

## 2. Dynamic Web Servers

A dynamic web server generates content at request time.

Instead of sending a pre-existing file, the server executes code and builds a response.

### Technologies

- PHP
- Python
- Node.js
- Java
- Ruby

### Example

A user requests:

```text
/profile
```

The server:

1. Queries a database
2. Retrieves user information
3. Generates HTML dynamically
4. Sends the personalized page

### Use Cases

- E-commerce websites
- Social media platforms
- Online banking
- Learning management systems
- APIs

### Advantages

- Personalized content
- Database integration
- Interactive functionality

### Disadvantages

- More complex
- Higher resource usage
- Increased security considerations

# HTTP vs HTTPS

HTTP:

```text
http://IP-ADDRESS
```

HTTPS:

```text
https://IP-ADDRESS
```

A fresh Apache installation usually supports HTTP first. HTTPS requires SSL/TLS certificates and additional configuration.

# The Document Root

The **document root** is the directory from which Apache serves files.

Ubuntu's default document root:

```text
/var/www/html
```

When users visit:

```text
http://server-address
```

Apache serves content from this directory.

---

# Default Web Pages

Apache looks for a default file named:

```text
index.html
```

For example:

```text
http://example.com/
```

is equivalent to:

```text
http://example.com/index.html
```

---

# Creating Your First Web Page

Navigate to the document root:

```bash
cd /var/www/html/
```

Backup the default page:

```bash
sudo mv index.html index.original.html
```

Create a new page:

```bash
sudo nano index.html
```

Add the following content:

```html
<html>
<head>
<title>My first web page using Apache</title>
</head>
<body>

<h1>Welcome</h1>

<p>
Welcome to my web site.
I created this site using the Apache HTTP server.
</p>

</body>
</html>
```

Save the file and refresh the browser.

---

# Viewing the Original Page

The backup page remains accessible.

Example:

```text
http://55.222.55.222/index.original.html
```

This demonstrates that browsers are simply requesting specific resources from the server.
